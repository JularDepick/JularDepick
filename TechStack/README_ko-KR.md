![C++](https://img.shields.io/badge/C%2B%2B-00599C?style=flat&logo=cplusplus&logoColor=white)
![PHP](https://img.shields.io/badge/PHP-777BB4?style=flat&logo=php&logoColor=white)

![Rust](https://img.shields.io/badge/Rust-333?style=flat&logo=rust&logoColor=white)

![Python](https://img.shields.io/badge/Python-3776AB?style=flat&logo=python&logoColor=white)

- C++는 강력한 컴파일 언어로, 객체 지향, 제네릭, 저수준 메모리 조작을 지원하며 시스템 개발, 게임 엔진, 고성능 연산에 널리 사용됨, 학습 곡선이 가파름
- PHP는 웹 개발을 위해 설계된 스크립트 언어로, 문법이 간단하여 배우기 쉬우며 Laravel 등 성숙한 프레임워크를 보유함, 주로 서버 사이드 웹 렌더링과 CMS 개발에 사용되며 진입 장벽이 낮음
- Node.js는 V8 엔진 기반의 JavaScript 런타임으로, 이벤트驱动와 논블로킹 I/O 모델을 채택하여 고并发 네트워크 애플리케이션과 API 서비스 구축에 적합함, 프론트엔드 개발자가 빠르게 시작할 수 있음
- TypeScript는 JavaScript의 상위 집합으로, 정적 타입 시스템과 인터페이스 등의 기능을 도입하여 대규모 프로젝트의 유지보수성을 향상시킴, 프론트엔드와 Node.js 백엔드 개발에 널리 사용되며 일정 수준의 타입 시스템 이해가 필요함
- Java는 크로스 플랫폼 객체 지향 언어로, 방대한 생태계와 엔터프라이즈급 프레임워크를 보유함, 주로 대규모 백엔드 시스템, 안드로이드 개발, 마이크로서비스 아키텍처에 사용되며 문법이 엄격하지만 학습 자원이 풍부함
- Rust는 메모리 안전을 중시하는 시스템 수준 언어로, 소유권 메커니즘을 통해 제로 비용 추상을 실현함, 시스템 프로그래밍, 임베디드, WebAssembly에 적합하며 학습 난이도가 높지만 안전성이 뛰어남
- Go는 Google에서 개발한 정적 타입 언어로, 간결한 문법과 내장 동시성 지원으로 유명함, 주로 클라우드 네이티브 서비스, 마이크로서비스, DevOps 도구 개발에 사용되며 배우기 쉬움
- C#는 마이크로소프트가 주도하는 객체 지향 언어로, .NET 생태계와 깊게 연동됨, Windows 애플리케이션, 게임 개발(Unity), 엔터프라이즈급 백엔드에 적합하며 Java와 유사하지만 더 현대적인 문법을 가짐
- Python은 동적 타입의 인터프리터 언어로, 문법이 간결하고 명료하며 데이터 과학, 인공지능, 자동화 스크립트, 웹 백엔드에 널리 사용됨, 입문은 쉽지만 전문가가 되려면 도메인 지식이 필요함


## 프론트엔드 언어

![HTML5](https://img.shields.io/badge/HTML5-E34F26?style=flat&logo=html5&logoColor=white)
![CSS](https://img.shields.io/badge/CSS-1572B6?style=flat&logo=css&logoColor=white)
![JavaScript](https://img.shields.io/badge/JavaScript-F7DF1E?style=flat&logo=javascript&logoColor=white)

- HTML5는 최신 HTML 표준으로, 시맨틱 태그, 오디오/비디오 재생, Canvas 드로잉을 지원하며 모든 웹 페이지의 구조적 기반임, 입문은 쉽지만 태그 시맨틱에 익숙해질 필요가 있음
- CSS는 웹 페이지 스타일을 기술하는 스타일시트 언어로, Flexbox, Grid 레이아웃, 애니메이션 효과를 지원하며 페이지의 시각적 표현을 담당함, 기초는 쉽게 배울 수 있지만 레이아웃과 반응형 디자인을 깊게 다루려면 실습이 필요함
- JavaScript는 웹의 유일한 네이티브 스크립트 언어로, 이벤트 처리, DOM 조작, 비동기 프로그래밍을 지원하며 웹 페이지의 인터랙션 로직 구현에 사용됨, 문법이 유연하지만 비동기와 프로토타입 체인 등의 개념을 이해해야 함


## 소프트웨어 및 도구

![Git](https://img.shields.io/badge/Git-F05032?style=flat&logo=git&logoColor=white)
![Nginx](https://img.shields.io/badge/Nginx-009639?style=flat&logo=nginx&logoColor=white)
![MySQL](https://img.shields.io/badge/MySQL-4479A1?style=flat&logo=mysql&logoColor=white)
![Redis](https://img.shields.io/badge/Redis-DC382D?style=flat&logo=redis&logoColor=white)
![SQLite](https://img.shields.io/badge/SQLite-003B57?style=flat&logo=sqlite&logoColor=white)
![Docker](https://img.shields.io/badge/Docker-2496ED?style=flat&logo=docker&logoColor=white)
![GitHub Actions](https://img.shields.io/badge/GitHub%20Actions-2088FF?style=flat&logo=githubactions&logoColor=white)

- Git은 분산 버전 관리 시스템으로, 브랜치 관리와 오프라인 커밋을 지원하며 코드 협업과 버전 관리의 사실상 표준임, 기본 조작은 간단하지만 고급 사용법(rebase, bisect)은 경험 축적이 필요함
- Nginx는 고성능 HTTP 서버 및 리버스 프록시로, 이벤트驱动 아키텍처를 통해 낮은 메모리 사용량과 높은 동시성을 실현함, 주로 정적 리소스 호스팅, 로드 밸런싱, 리버스 프록시에 사용되며 설정 문법에 익숙해질 필요가 있음
- MySQL은 가장 인기 있는 오픈소스 관계형 데이터베이스로, 트랜잭션, 인덱스, 마스터-슬레이브 복제를 지원하며 웹 애플리케이션의 데이터 저장소에 널리 사용됨, 기본 CRUD는 간단하지만 성능 튜닝은 경험을 필요로 함
- Redis는 메모리 기반의 키-값 데이터베이스로, 문자열, 해시, 리스트 등 다양한 자료구조와 발행-구독을 지원함, 주로 캐시, 세션 관리, 메시지 큐에 사용되며 명령어는 간단하지만 아키텍처 설계를 이해해야 함
- SQLite는 경량 임베디드 관계형 데이터베이스로, 별도의 서버 프로세스가 필요 없으며 데이터를 단일 파일로 저장함, 모바일, 임베디드, 소규모 애플리케이션에 적합하며 사용이极其 간단함
- Docker는 컨테이너화 플랫폼으로, 이미지와 컨테이너를 통해 애플리케이션의 패키징과 격리 배포를 실현함, 주로 환경 일관성 보장과 마이크로서비스 배포에 사용되며 기본 명령어는 쉽게 배울 수 있지만 오케스트레이션은 실습이 필요함
- GitHub Actions는 GitHub 내장 CI/CD 플랫폼으로, YAML 워크플로우를 통해 자동화된 빌드, 테스트, 배포를 구현함, GitHub 저장소와 깊게 통합되며 설정 문법이 직관적이고 배우기 쉬움


## 개발 프레임워크

![Spring Boot](https://img.shields.io/badge/Spring%20Boot-6DB33F?style=flat&logo=springboot&logoColor=white)
![Vue.js](https://img.shields.io/badge/Vue.js-4FC08D?style=flat&logo=vuedotjs&logoColor=white)
![React](https://img.shields.io/badge/React-61DAFB?style=flat&logo=react&logoColor=white)

- Spring Boot는 Spring 기반의 빠른 개발 프레임워크로, 자동 설정과 관례 우선 설정을 통해 엔터프라이즈급 애플리케이션 개발을 단순화함, 주로 Java 백엔드 마이크로서비스와 RESTful API에 사용되며一定的 수준의 Spring 기초 지식이 필요함
- Vue.js는 점진적 프론트엔드 프레임워크로, 반응형 데이터 바인딩과 컴포넌트 기반 설계를 채택하여 기존 프로젝트와의 통합이 용이함, 싱글 페이지 애플리케이션과 중소규모 프로젝트에 널리 사용되며 접근성이 좋고 문서가 잘 갖춰져 있음
- React는 Facebook에서 개발한 프론트엔드 UI 라이브러리로, JSX와 가상 DOM을 기반으로 효율적인 렌더링을 실현함, 방대한 생태계(Next.js, React Native)를 보유하고 있으며 대규모 애플리케이션에 적합하지만 함수형 프로그래밍 사고를 이해해야 함
